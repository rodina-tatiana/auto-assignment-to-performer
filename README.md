# Auto Assignee — Yandex Tracker

Функция для Yandex Cloud Functions, которая автоматически назначает исполнителя на задачу в Yandex Tracker. Находит участника команды с наименьшим количеством активных задач и назначает задачу на него.

---

## 1. Требования

- Python 3.10+
- Библиотека `requests`
- Yandex Cloud Functions
- OAuth-токен с доступом к Yandex Tracker

---

## 2. Как работает

1. Триггер в Yandex Tracker отправляет HTTP-запрос с ключом задачи на функцию
2. Функция получает список участников команды из указанной очереди
3. Для каждого участника считается количество активных задач через Tracker Search API
4. Выбирается участник с наименьшим количеством задач
5. Задача назначается на этого участника
6. В логах выводится статистика по загрузке команды

---

## 3. Как добавить пользователей в команду очереди

Именно из этого списка функция берёт участников для назначения.

1. Открыть очередь в Yandex Tracker
2. Перейти в **Настройки очереди → Участники**
3. Добавить нужных пользователей в раздел **Команда**
4. Указать ключ этой очереди в переменной окружения `TRACKER_QUEUE`

---

## 4. Как настроить триггер в Yandex Tracker

Триггер отправляет ключ задачи в функцию при наступлении нужного события.

1. Открыть очередь в Yandex Tracker
2. Перейти в **Настройки очереди → Триггеры → Создать триггер**
3. Настроить условие срабатывания — например, **Статус задачи изменился на "Открыт"**
4. В разделе **Действия** выбрать **HTTP-запрос** и заполнить:

- **Метод:** `POST`
- **URL:** ссылка на вашу функцию в Yandex Cloud Functions
- **Тело запроса:**
```json
{
  "issueKey": "{{issue.key}}"
}
```
- **Заголовок:** `Content-Type: application/json`

5. Сохранить триггер

> `{{issue.key}}` — это шаблонная переменная Tracker, которая автоматически подставляет ключ задачи при срабатывании триггера

---

## 5. Переменные окружения

Задаются в настройках функции в Yandex Cloud Functions.

**Обязательные:**

- `TRACKER_TOKEN` — OAuth-токен для доступа к Tracker API
- `TRACKER_ORG_ID` — ID организации в Yandex Tracker

**Необязательные:**

- `TRACKER_ORG_HEADER` — заголовок для передачи ID организации. По умолчанию `X-Org-ID`. Для Yandex Cloud организаций указать `X-Cloud-Org-ID`
- `TRACKER_QUEUE` — ключ очереди, из которой берётся список команды. По умолчанию `PM`
- `STATUS_TYPES` — типы статусов задач через запятую, которые учитываются при подсчёте загрузки. Например: `new,inProgress`. Если не задано — считаются все задачи без фильтра по статусу
- `DEBUG` — подробные логи. Значения: `1`, `true`, `yes`, `on`. По умолчанию выключен

---

## 6. Логи в консоли

После каждого вызова функция выводит результат в консоль Yandex Cloud Functions:

```
Задача "PM-123" назначена пользователю 8000000000000510

Статистика по загрузке:
  Пользователь 8000000000000510 — 114 задач ←
  Пользователь 8000000000000511 — 115 задач
  Пользователь 8000000000000512 — 118 задач
```

Стрелка `←` указывает на выбранного исполнителя.

При включённом `DEBUG=1` дополнительно выводятся:
- поисковый запрос к Tracker API для каждого пользователя
- полные заголовки ответа
- тело ответа от API

## 7. Возможные ошибки

**В очереди не найдены участники команды**

Функция вернёт ошибку `500` и сообщение `No teamUsers with ids found`.

Что сделать: проверить, что в очереди `TRACKER_QUEUE` добавлены участники в раздел **Команда** (см. раздел 3).

---

**Ошибка Tracker API со статусом 401**

Что сделать: проверить, что `TRACKER_TOKEN` указан верно и токен не истёк.

---

**Ошибка Tracker API со статусом 403**

Что сделать: проверить, что токен имеет доступ к нужной очереди и право на редактирование задач.

---

**Ошибка Tracker API со статусом 404**

Что сделать: проверить, что ключ очереди в `TRACKER_QUEUE` указан верно и такая очередь существует.

---

**В логах нет заголовка `X-Total-Count`**

Функция выбросит ошибку `X-Total-Count header is missing in Tracker response`.

Что сделать: включить `DEBUG=1` и проверить полные заголовки ответа от Tracker API. Возможно, изменился формат ответа API.

---

**Задача назначается одному и тому же человеку при параллельных вызовах**

Это известное ограничение — при одновременных вызовах функции между подсчётом задач и назначением состояние может измениться.

Что сделать: настроить триггер так, чтобы он не срабатывал слишком часто, или добавить задержку между вызовами.
